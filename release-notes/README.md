# Release Notes

This directory contains release notes for each tagged version of Maravilla Konto.

## Convention

- Create a Markdown file named after the git tag, e.g. `v0.1.0-beta.1.md`
- The GitHub Actions release workflow uses the matching file as the release body
- If no file exists for a tag, a default message is used
- Keep notes concise — they appear in the app's auto-update dialog and on GitHub Releases
