# Security & Reliability Fixes

This release addresses 5 issues (2x P1, 3x P2) found during code review, improving auth flow reliability, backend validation, and error handling.

## Fixes

- **Auth refresh deadlock** — Token refresh now uses a dedicated HTTP client without interceptors, preventing request loops when the access token expires
- **Corrupted localStorage handling** — Malformed `konto_tokens` in localStorage no longer crashes the app; invalid JSON is cleared and the user is redirected to login
- **Rate limiter memory leak** — Expired rate-limit entries are now fully pruned on each check, preventing unbounded HashMap growth
- **File upload content validation** — Uploads now verify file magic bytes (PNG, JPEG, WebP, GIF, PDF) in addition to filename extension, rejecting spoofed file types
- **Panic-free runtime code** — Replaced `.unwrap()` calls in fiscal year setup and PDF invoice generation with proper error propagation
